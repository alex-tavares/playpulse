# RFC-007 - Godot SDK Surface & Reliability Guarantees

**Status**: Ratified and implemented  
**Owners**: SDK Lead, Tech Lead  
**Reviewers**: Backend Engineer, Privacy Officer, QA Lead

## Context & Goals
- Define the public API surface for the PlayPulse Godot 4 SDK.  
- Establish queueing, batching, and backoff strategies aligned with ingest limits.  
- Document performance budgets and failure behaviors so game teams know what to expect.

## API Surface
SDK is delivered as a Godot autoload (`PlayPulse`) initialized during game boot.

| Method | Purpose | Notes |
| --- | --- | --- |
| `configure(config: Dictionary)` | Initialize SDK with credentials and runtime options. | Trusted HMAC mode requires `api_key`, `signing_secret`, `game_id`, `game_version`, `build_id`, and `ingest_base_url`. Public client mode requires `auth_mode = "public_client"`, `client_id`, `platform_channel`, `game_id`, `game_version`, `build_id`, and `ingest_base_url`. Optional: `player_seed`, `locale`, `initial_consent` (default `true`), `flush_interval_sec` (default 5). Generates/stores `device_id`, derives `player_id_hash`, opens persistence store, schedules flush timer. No events accepted before configure succeeds. |
| `track(event_name: String, props: Dictionary = {})` | Enqueue event for delivery. | Auto-populates envelope (`event_id`, timestamps, session_id, schema_version, consent flag). Validates property sizes/types against RFC-002 caps. Returns `OK` or error code (`ERR_NOT_CONFIGURED`, `ERR_CONSENT_DISABLED`, `ERR_QUEUE_FULL`). |
| `flush(force: bool = false)` | Trigger immediate send of queued events. | Sends batches up to 10 events. If `force`, bypasses the temporary circuit-open guard after terminal 4xx failures. Returns `OK`, `ERR_BUSY`, or a transport error code and emits `flush_completed(success: bool, metadata: Dictionary)`. |
| `set_consent(enabled: bool)` | Toggle analytics consent. | Setting to `false` clears queue and persistence, future `track` no-ops until re-enabled. |
| `set_player_identifier(seed: String)` | Re-derive `player_id_hash`. | Useful when a user logs into an account; pending queue flushed before swap. |
| `shutdown()` | Flush outstanding events and close persistence. | Appends `session_end`, attempts a best-effort flush, and leaves unsent events persisted for replay on the next launch. |

Internal helpers (not part of public API) handle session lifecycle (`session_id` rotation), persistence, signing, and transport diagnostics.

## Queue & Transport Strategy
- **In-memory queue cap**: 200 events. When length >= 10, enqueue task schedules immediate flush.  
- **Flush interval**: default 5 s (configurable 2-30 s). Timer triggers flush even if queue < batch size to keep latency low.  
- **Batch size**: up to 10 events per HTTP POST. Matches ingest expectations (`POST /events`).  
- **Offline persistence**: append-only JSONL stored under `user://playpulse/events.log`. Capped at 2,000 events or 2 MB. Oldest entries dropped when limit exceeded (warning logged). On startup, persisted events replayed before new ones.  
- **Backoff**: exponential with jitter per batch - 1 s, 5 s, 15 s, 60 s, 60 s (max). After five failed attempts, the queue remains persisted for a later retry.  
- **Signing**: HMAC signs the canonical string `"{timestamp}\n{nonce}\n{raw_body}"` and sends `X-Api-Key`, `X-Signature`, `X-Request-Timestamp`, and `X-Nonce` headers to ingest.  
- **Compression**: deferred from the MVP implementation. Current SDK sends JSON bodies uncompressed to match the documented ingest behavior.  
- **Heartbeat diagnostics**: deferred from the MVP implementation.  
- **Session keep-alive**: `session_id` renewed after 30 minutes of inactivity or on explicit `session_start` event.

## Performance Budget & Failure Behavior
- **CPU**: `track` call does property validation + enqueue on the main thread. JSON serialization + HMAC signing are lightweight and stay on the main thread in the MVP implementation.  
- **Memory**: queue + metadata < 512 KB; persistence file limited to 2 MB.  
- **Network**: uses Godot `HTTPRequest` with async callback; main thread not blocked.  
- **Consent disabled**: all `track` calls ignored (returns `ERR_CONSENT_DISABLED`). Queue cleared in-memory and on disk.  
- **4xx responses** (auth, rate limit, schema): batch dropped, error logged, circuit opened for 60 s to avoid spam.  
- **5xx/timeouts**: follow backoff strategy; persisted after retries exhausted.  
- **Disk write failures**: fallback to memory-only mode; if queue overflows, newest events dropped with warning.  
- **Shutdown**: `shutdown()` attempts a best-effort flush and relies on persistence replay instead of a hard synchronous wait loop.  
- **Telemetry diagnostics**: queue-length counters and heartbeat alerts are deferred beyond the MVP implementation.

## Example Usage (Godot 4 GDScript)
```gdscript
extends Node

func _ready():
    PlayPulse.configure({
        "api_key": OS.get_environment("PLAYPULSE_GODOT_API_KEY"),
        "signing_secret": OS.get_environment("PLAYPULSE_GODOT_SIGNING_SECRET"),
        "game_id": "mythtag",
        "game_version": ProjectSettings.get_setting("application/config/version", "0.0.0"),
        "build_id": "mythtag-local",
        "ingest_base_url": OS.get_environment("PLAYPULSE_GODOT_INGEST_BASE_URL"),
        "player_seed": OS.get_environment("PLAYPULSE_GODOT_PLAYER_SEED"),
        "locale": TranslationServer.get_locale(),
        "initial_consent": true
    })
    PlayPulse.track("session_start", {
        "launch_reason": "fresh_launch",
        "connection_mode": "online"
    })

func start_match(match_id: String):
    PlayPulse.track("match_start", {
        "match_id": match_id,
        "mode_id": "arena_ranked",
        "team_size": 3
    })

func _notification(what):
    if what == NOTIFICATION_WM_CLOSE_REQUEST:
        PlayPulse.shutdown()
        get_tree().quit()
```

## Local Validation Flow
- SDK source lives under `sdk/godot/playpulse/addons/playpulse`.
- Use `scripts/godot/Run-SdkTests.ps1` to sync the addon into the local Godot test project and run the automated SDK suite with the Godot console executable.
- Game-specific bridges and local consumer validation helpers now live in the companion examples repo: [alex-tavares/playpulse-examples](https://github.com/alex-tavares/playpulse-examples).
- The shipped SDK remains generic and game-agnostic; examples are kept outside the core repo.

## Deferred Beyond MVP
- Runtime-adjustable flush intervals beyond `configure`
- Built-in event sampling
- Gzip request compression
- Heartbeat / internal telemetry counters
- C# bindings and additional Godot export targets

## Decision
Adopt the above SDK surface, queue strategy, and reliability guarantees for the Godot 4 MVP. The addon and standalone SDK tests remain implemented in the core repository; game-specific bridges are maintained in the companion examples repo.

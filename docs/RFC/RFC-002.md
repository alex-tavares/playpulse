# RFC-002 - Telemetry Event Candidates

**Status**: Draft for review  
**Owners**: Product Manager, Tech Lead (PlayPulse)  
**Reviewers**: Ingest Lead, Privacy/Security, Capivarious pilot devs

## Context & Goals
- Define the initial telemetry surface for the Ingest MVP without locking later roadmap events.
- Support pilot titles MythClash and MythTag with events that cover sessions, matches, and character engagement.
- Ensure every proposed field honors PlayPulse privacy guardrails and is feasible for the Godot SDK to emit.

## Event Envelope (shared fields)

All events include the following keys, emitted by the SDK and enforced at ingest:

| Field | Type | Notes and Limits |
| --- | --- | --- |
| `event_id` | UUID v4 | SDK-generated per event. |
| `event_name` | string | `snake_case`, max 48 chars. |
| `schema_version` | string (`major.minor`) | Starts at `1.0`; additive changes bump minor, breaking changes bump major. |
| `occurred_at` | ISO8601 string | Client timestamp in UTC. |
| `session_id` | UUID v4 | Stable for duration of session; rotates on `session_start`. |
| `player_id_hash` | 64-char hex string | Salted SHA-256 hash; never raw IDs. |
| `game_id` | enum | `mythclash`, `mythtag`. |
| `game_version` | semver string | Example: `0.6.2`. |
| `build_id` | short string | Max 16 chars. |
| `platform` | enum | `pc`, `mac`, `linux`. |
| `locale` | IETF BCP 47 string | Optional, max 8 chars. |
| `consent_analytics` | boolean | `false` data is stored but excluded from analytics reads. |
| `properties` | object | Event-specific payload capped at 1.5 KB uncompressed. |

### Global limits
- Event payload (envelope + properties) must remain under 2 KB.
- Arrays are capped at 10 elements.
- Strings are capped at 64 characters unless noted otherwise.
- Numeric values are integers unless explicitly flagged as decimal.

## MVP Event Catalog

### `session_start`
- `launch_reason` (enum): `fresh_launch`, `resume`, `hot_reload`
- `connection_mode` (enum): `online`, `offline`
- `timezone_offset_min` (integer): range `-720` to `840`

### `session_end`
- `duration_s` (integer): `0` to `86400`
- `exit_reason` (enum): `user_exit`, `disconnect`, `crash`, `idle_timeout`
- `xp_earned` (integer): `0` to `1000`

### `match_start`
- `match_id` (UUID v4): generated client-side per match
- `mode_id` (string): max 32 chars
- `map_id` (string): max 32 chars
- `team_size` (integer): `1` to `5`
- `party_size` (integer): `1` to `4`
- `mmr_bucket` (enum): `bronze`, `silver`, `gold`, `diamond`

### `match_end`
- `match_id` (UUID v4): must match `match_start`
- `duration_s` (integer): `0` to `7200`
- `result` (enum): `win`, `loss`, `draw`, `abandon`
- `character_id` (string): max 32 chars
- `score` (integer): `0` to `100000`
- `damage_dealt` (integer): `0` to `500000`

### `character_selected`
- `selection_context` (enum): `match_lobby`, `armory`, `tutorial`
- `character_id` (string): max 32 chars
- `loadout_id` (string): optional, max 32 chars
- `perk_ids` (array<string>): max 5 entries, each max 32 chars
- `is_random` (boolean)

## Deferred / Backlog Events

These are not part of the MVP contract:

- `tutorial_step_completed`
- `client_performance_sample`
- `error_raised`
- `store_item_purchased`

## Custom Event Extensions (post-MVP)

- SDK support for `custom.<game_id>.<event_key>` is deferred to post-MVP.
- Custom schema registration, schema hash publishing, and quarantine handling remain backlog work.
- Analytics consumers must not assume custom events are available in the MVP analytics surface.

## Versioning & Evolution Policy

- Initial release uses `schema_version = 1.0`.
- Additive fields to existing events bump the minor version.
- New event types reuse the current major version.
- Removing or renaming fields, or altering types, bumps the major version.
- MVP ingest accepts only major version `1`.

## Privacy Check

- No raw identifiers, emails, IPs, chat logs, or free-text fields are collected.
- Hashing ensures `player_id_hash` cannot be reversed without a salt.
- Enums and bounded integers reduce leakage risk.
- `timezone_offset_min` stores only numeric offset, not a timezone name.
- `consent_analytics` is retained to honor opt-out across aggregation layers.

## Remaining Open Questions

1. Hashing strategy: Should the salt be per title or per device install?
2. Offline buffering: What is the maximum offline queue length or age before drops occur?
3. Match identifiers: Should the server ever override client-supplied `match_id` in future multiplayer modes?
4. Schema enforcement: Must the SDK preserve canonical key ordering for HMAC signing, or may ingest normalize internally?
5. Locale capture: Should `locale` remain optional in the pilot dataset?

## Decision

Use this RFC as the candidate history for the finalized MVP event set documented in `docs/EVENTS.md`. Future additions beyond the five-event MVP require a follow-up RFC or decision-log update.
